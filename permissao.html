Permissões no Linux

No Linux, as permissões de arquivos e diretórios são um aspecto fundamental da segurança e do controle de acesso. As permissões determinam quem pode ler, escrever ou executar um arquivo ou diretório. Aqui está uma visão geral das permissões no Linux:

Tipos de Permissões

  • Leitura (r): Permite que o usuário leia o conteúdo do arquivo ou liste o conteúdo de um diretório.
  • Escrita (w): Permite que o usuário modifique o conteúdo do arquivo ou adicione/remova arquivos em um diretório.
  • Execução (x): Permite que o usuário execute um arquivo como um programa ou acesse um diretório.

Classes de Usuários

  • Usuário (u): O proprietário do arquivo.
  • Grupo (g): Os usuários que pertencem ao mesmo grupo que o proprietário do arquivo.
  • Outros (o): Todos os outros usuários que não são o proprietário nem pertencem ao grupo.

Visualizando Permissões

Você pode visualizar as permissões de arquivos e diretórios usando o comando ls -l. A saída será algo como:

-rwxr-xr-- 1 user group 1234 Jan 01 12:00 arquivo.txt

A primeira coluna indica as permissões:

  • O primeiro caractere indica o tipo de arquivo (- para arquivo regular, d para diretório, etc.).
  • Os próximos nove caracteres são divididos em três grupos de três:
    • O primeiro grupo (posições 2-4) mostra as permissões do usuário (u).
    • O segundo grupo (posições 5-7) mostra as permissões do grupo (g).
    • O terceiro grupo (posições 8-10) mostra as permissões para outros (o).

Modificando Permissões

Você pode modificar as permissões usando o comando chmod. Existem duas maneiras de usar o chmod: a notação simbólica e a notação octal.

Notação Simbólica

  • Para adicionar uma permissão: chmod u+x arquivo.txt (adiciona permissão de execução para o usuário).
  • Para remover uma permissão: chmod g-w arquivo.txt (remove permissão de escrita para o grupo).

Notação Octal

As permissões são representadas por números:

  • Leitura = 4
  • Escrita = 2
  • Execução = 1

Você pode somar esses valores para definir permissões. Por exemplo, chmod 755 arquivo.txt define:

  • Usuário: leitura, escrita e execução (4+2+1=7)
  • Grupo: leitura e execução (4+1=5)
  • Outros: leitura e execução (4+1=5)
